**CloudWatch** is a **monitoring and observability** service provided by AWS. It lets you monitor your AWS resources and applications in real-time.

## ğŸ§  **Key Concepts in CloudWatch**

### 1. **Metrics**
- These are **time-series data points**.
- Example: CPU usage of an EC2 instance, number of requests to an API Gateway.
- AWS services send default metrics, and you can also publish **custom metrics**.

### 2. **Logs**
- You can send logs (from EC2, Lambda, ECS, etc.) to CloudWatch.
- These logs can be searched, filtered, and analyzed.
- Useful for debugging and auditing.

### 3. **Alarms**
- You can set thresholds on metrics.
- If a metric crosses the threshold, CloudWatch can:
  - Send an email via SNS
  - Trigger an auto-scaling policy
  - Invoke a Lambda function
- Example: Send an alert if CPU usage > 80% for 5 minutes.

### 4. **Dashboards**
- Create visual dashboards using widgets like graphs, numbers, etc.
- Monitor your system visually in one place.

### 5. **Events / EventBridge**
- Earlier known as **CloudWatch Events**.
- It helps you **respond automatically** to system events.
- Example: Trigger a Lambda when an EC2 instance is terminated.
