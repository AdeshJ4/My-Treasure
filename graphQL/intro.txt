ðŸŸ  Differences from REST:

ðŸ‘‰ Flexible Data Fetching â€“ Clients specify the exact fields they need, unlike REST, where fixed endpoints return predefined data.

ðŸ‘‰ Single Endpoint â€“ GraphQL typically uses a single "/graphql" endpoint, whereas REST has multiple endpoints for different resources.

ðŸ‘‰ Efficient Data Fetching â€“ Fetch multiple related resources in a single request, avoiding multiple API calls in REST.

ðŸ‘‰ Strongly Typed Schema â€“ GraphQL uses a "schema" to define the structure of API responses, while REST relies on standard JSON responses.

ðŸ‘‰ Backend Agnostic â€“ Works with any database or programming language, making it highly adaptable.

ðŸ”¹ GraphQL on the Backend
ðŸ“Œ The backend defines the GraphQL schema and resolves queries.

The GraphQL resolver functions fetch data from databases or APIs.

Example: Backend (Node.js with Apollo Server)
```
const { ApolloServer, gql } = require('apollo-server');

const typeDefs = gql`
  type Movie {
    id: ID!
    title: String!
    director: String!
  }

  type Query {
    movie(id: ID!): Movie
  }
`;

const resolvers = {
  Query: {
    movie: (_, { id }) => ({ id, title: "Inception", director: "Christopher Nolan" })
  }
};

const server = new ApolloServer({ typeDefs, resolvers });
server.listen().then(({ url }) => console.log(`ðŸš€ Server ready at ${url}`));
```


ðŸ”¹ GraphQL on the Client
ðŸ“Œ The client sends GraphQL queries to the backend.
Uses GraphQL client libraries like Apollo Client, Relay, or urql.
Fetches only the required data.

Example: Client (React with Apollo Client)
```
import { gql, useQuery } from '@apollo/client';

const GET_MOVIE = gql`
  query GetMovie($id: ID!) {
    movie(id: $id) {
      title
      director
    }
  }
`;

function MovieDetails({ id }) {
  const { loading, error, data } = useQuery(GET_MOVIE, { variables: { id } });

  if (loading) return <p>Loading...</p>;
  if (error) return <p>Error: {error.message}</p>;

  return <h1>{data.movie.title} - Directed by {data.movie.director}</h1>;
}
```

â“ Explain the difference between "queries", "mutations", and "subscriptions" in GraphQL.

1ï¸âƒ£ Queries â€“ Fetch Data

Similar to GET requests in REST.
Used to retrieve data without modifying it.
Clients can request specific fields.

Example:
  Query: {
    movies: async () => await Movie.find(),
    movie: async (_, { id }) => await Movie.findById(id),
  },



2ï¸âƒ£ Mutations â€“ Modify Data

Similar to POST, PUT, DELETE in REST.
Used to create, update, or delete data.
Can return updated data after execution.

Example:
Mutation: {
    addMovie: async (_, { title, director, releaseYear }) => {
      const movie = new Movie({ title, director, releaseYear });
      return await movie.save();
    },
    updateMovie: async (_, { id, title, director, releaseYear }) => {
      return await Movie.findByIdAndUpdate(
        id,
        { title, director, releaseYear },
        { new: true }
      );
    },
    deleteMovie: async (_, { id }) => {
      await Movie.findByIdAndDelete(id);
      return "Movie deleted successfully";
    },
  },


3ï¸âƒ£ Subscriptions â€“ Real-time Updates

Uses "WebSockets" to push real-time updates to clients.
Clients stay connected and get notified when data changes.
we can integrate GraphQL Subscriptions using WebSockets (graphql-ws).

Example:
subscription {
  movieAdded {
    id
    title
  }
}

Resolvers fetch data from a database, API, or other sources.
The returned data is structured according to the GraphQL schema
